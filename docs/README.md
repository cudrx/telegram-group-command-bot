# Структура Документации

Карта Markdown-документов в репозитории.

## Основные документы

- `README.md` — обзор проекта, быстрый старт, переменные окружения и команды.
- `docs/README.md` — эта карта документации.
- `docs/architecture.md` — архитектура, инварианты, основные потоки и модель данных.
- `docs/development.md` — локальная разработка, проверки, CI/CD, Docker и заметки по продакшну.
- `AGENTS.md` — точка входа с правилами для Codex и похожих coding agents.
- `agent/` — модульные инструкции для агентов, маршрутизация задач, правила разработки и playbook-файлы.
- `llm/` — статические prompt-файлы.
- `llm/assistant/base.md` — базовые инструкции ассистента.

## Правила

- Не плодить Markdown рядом с кодом без необходимости.
- Новые проектные документы, планы реализации, планы выката и описания задач хранить в `docs/superpowers/plans/`.
- Если новый документ не попадает в существующую структуру, сначала обновить этот файл.
- Предпочитать обновление существующего документа созданию почти дубля.
- Не ссылаться на локальные абсолютные пути машины разработчика.
