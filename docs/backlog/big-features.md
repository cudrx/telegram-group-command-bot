# Big Feature Backlog

Крупные подсистемы, которые нельзя возвращать пачкой. Каждую нужно проектировать, включать отдельно и проверять в проде поверх стабильного v0.

## Вырезанные post-v0 слои

- Idle summary: фоновая LLM-сводка чата после затихания.
- Summary-based retention: удаление старых сообщений только после покрытия summary.
- Participant memory: chat-scoped факты об участниках.
- Read-only explicit memory как первый безопасный возврат memory-layer.
- Inferred/observed memory: социальные и поведенческие выводы, только после read-only explicit memory.
- Participant aliases: chat-scoped aliases и deterministic participant resolution.
- Social-QA: ответы про участников, отношения, динамику и “что между X и Y”.
- Autonomous interjections: самостоятельные вмешательства без явного `@mention` или reply.
- `silence_after_activity`: таймерный сценарий оживления чата после паузы.
- Per-chat persona override через `config/personas/<chat_id>.md`.
- Режимы persona/mode для конкретного чата.
- Игровая мимикрия под участника.
- Manual LLM reply eval pack для текущего v0 prompt.

## Социальный интеллект

- Отслеживать групповую динамику, а не только отдельные факты об участниках: кто с кем шутит, кто кого поддерживает, какие пары и компании часто возникают.
- Отслеживать локальные мемы чата и мини-сюжеты, которые быстро появляются и исчезают.
- Добавить ручные заметки об участниках поверх автоматически собранных профилей.
- Добавить команду для краткой сводки по участнику и забавной статистики по чату.
- Добавить команду по запросу: “что произошло, пока нас не было”.
- Научить бота отмечать участников, когда это уместно в социальном контексте.

## Мультимодальность

- Читать изображения: понимать фото, мемы и скриншоты, а потом использовать этот контекст в ответах.
- Смотреть видео: извлекать суть из коротких роликов и реагировать на них в чате.
- Слушать аудио: понимать голосовые сообщения и короткие аудиофрагменты.
- Отправлять картинки: генерировать или подбирать визуальные ответы, когда это смешнее или выразительнее обычного текста.

## Эксперименты с персоной

- Добавить игровой режим “мимикрия под участника”.
- Добавить быстрое переключение persona/mode для конкретного чата.

## Более умное использование LLM

- Хранить снимки профилей во времени, чтобы уменьшить drift.
- Разбивать длинную историю чата на скользящие окна перед compression.
- После деплоя отправлять в бота краткое summary апдейта, чтобы runtime мог учитывать, что изменилось в новой версии.
