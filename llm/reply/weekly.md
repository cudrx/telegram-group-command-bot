Составь недельный дайджест чата по предоставленному WEEKLY_DATASET.

Пиши по-русски. Формат вывода: Telegram HTML. Разрешены только теги <b>, <i>, <code>. Не используй Markdown, списки с дефисами, таблицы, ссылки без явной опоры в данных или выдуманные цитаты.

Используй только факты из WEEK_STATS, PARTICIPANT_STATS и SELECTED_EVENTS. Не достраивай числовые отношения, причинно-следственные связи и мотивации участников, если они явно не следуют из данных. Если SELECTED_EVENTS пустой или слабый, честно сделай короткий спокойный обзор по статистике, без драматизации. Не называй пользователей по userId, если есть displayName. Не раскрывай внутренние поля, оценки score, messageIds, participantIds или причины отбора как технические данные.

Числа и сравнения:

- Не выводи производные числовые сравнения, если они явно не указаны в WEEK_STATS или PARTICIPANT_STATS.
- Не пиши формулировки вроде “в два раза больше”, “почти вдвое”, “на порядок больше”, “резко больше”, если такое сравнение уже не посчитано во входных данных.
- Если нужно сравнить активность участников, используй только прямые числа из PARTICIPANT_STATS: “Хачик — 416 сообщений, Артём — 385”.
- Не округляй отношения между числами самостоятельно.
- Не делай вывод о доминировании участника только по первому месту в статистике, если разрыв небольшой.

Обязательная структура:

<b>Неделя в чате</b>

• 2-3 пункта с главным ходом недели.

<b>Темы</b>

• Только темы, подтвержденные SELECTED_EVENTS.

<b>Моменты</b>

• Конкретные эпизоды недели, не общий пересказ настроения.

<b>Роли недели</b>

• Только роли, заметные по сообщениям этой недели.

<b>Факты</b>

• Короткие факты с опорой на данные недели.

<b>Итог</b>

Один короткий завершающий абзац.
